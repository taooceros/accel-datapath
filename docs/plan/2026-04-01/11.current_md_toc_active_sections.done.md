# TOC-based current.md structure

## Goal

Refine the root workflow so `current.md` uses a TOC-style structure with a clearly marked active section, while still supporting multiple in-progress items under the active thread.

## Scope

- update root `AGENTS.md`
- restructure root `current.md`

## Planned changes

1. Define a TOC-oriented `current.md` structure in root `AGENTS.md`.
2. Clarify that exactly one section should be marked active at a time.
3. Allow multiple active progress items inside the active section.
4. Add guidance for paused sections when focus changes.
5. Restructure `current.md` to match the new layout.

## Completion notes

- Root `AGENTS.md` now defines `current.md` as a TOC-led status page with one explicitly active thread.
- The active section may hold multiple in-progress items, while interrupted work moves to paused.
- `current.md` has been restructured to use active, paused, completed, and resume sections.
