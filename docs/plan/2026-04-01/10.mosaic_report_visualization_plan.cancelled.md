# Plan: Mosaic Integration for Report Visualization

**Date**: 2026-04-01

**Status**: cancelled on 2026-04-03; superseded by the Observable Framework rewrite plan.

## Goal

Integrate `uwdata/mosaic` into the tonic profiling report visualization flow so the bounded-matrix results can be explored interactively from a report artifact rather than only through static markdown tables.

## Scope

### In scope

- add a small local web visualization built on Mosaic/VGPlot
- target the bounded tonic matrix results in `results/tonic/2026-04-01-loop2/matrix_summary.csv`
- expose linked filters/views for the core experiment dimensions and metrics
- attach the generated visualization to the existing benchmarking report as a report artifact

### Out of scope

- replacing all existing Plotly/Matplotlib tooling across the repo
- building a general-purpose report web app
- changing the tonic benchmark schema itself

## Proposed integration path

1. Create a small Mosaic app under `tools/` with its own `package.json` and build script.
2. Use DuckDB-WASM through Mosaic's browser connector so the dashboard remains local and static-hostable.
3. Load the tonic `matrix_summary.csv` directly and derive human-readable labels inside the app.
4. Build the artifact into `docs/report/artifacts/` and link it from `docs/report/benchmarking/009.tonic_profile_bounded_matrix_results.md`.

## Intended views

- controls for payload kind, compression, runtime, and metric selection
- throughput vs concurrency by payload size
- latency vs concurrency by payload size
- a detail table filtered by linked selections

## Verification

- install dependencies and produce a local build successfully
- verify the built artifact references the expected CSV path
- run diagnostics on edited source files where available

## Deliverables

- Mosaic app source under `tools/`
- built artifact under `docs/report/artifacts/`
- report update linking the new interactive visualization
