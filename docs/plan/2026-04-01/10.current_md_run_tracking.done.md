# Current.md run tracking policy

## Goal

Strengthen the root workflow so `current.md` acts as the active run ledger: it should carry the overall goal for the current run and be updated immediately as individual tasks complete.

## Scope

- update root `AGENTS.md`
- clarify how `current.md` should be used during active work and on resume

## Planned changes

1. Add explicit `current.md` workflow guidance to root `AGENTS.md`.
2. Require an overall goal for the current run to be maintained in `current.md`.
3. Require individual tasks to be marked complete in `current.md` immediately when they finish.
4. Clarify that `current.md` is an active status ledger, not just a passive resume note.

## Completion notes

- Root `AGENTS.md` now defines a dedicated `current.md` workflow section.
- The policy now requires an explicit overall goal for the current run in `current.md`.
- The policy now requires individual items to be marked complete in `current.md` immediately when they finish.
- The conventions section now requires `current.md` to be brought up to date before non-trivial work continues.
