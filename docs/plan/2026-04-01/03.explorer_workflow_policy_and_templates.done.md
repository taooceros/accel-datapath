# Explorer workflow policy and templates

## Goal

Add durable repo guidance and reusable agent assets so `@explorer` stays bounded, returns early, and avoids context blowups that trigger auto compaction.

## Problem

- `@explorer` tasks are easy to overscope.
- Discovery, deep reading, and synthesis were too easy to combine in one delegation.
- Missing prompt-shaping and stop-condition defaults made it easy for an explorer run to retain too many candidates and intermediate notes.

## Scope

- tighten root `AGENTS.md` explorer policy
- add a dedicated explorer workflow guide under `.agents/workflows/`
- add reusable prompt templates under `.agents/templates/`
- keep the policy short in `AGENTS.md` and move procedural detail into `.agents/`

## Planned changes

1. Replace the broad explorer workflow bullets in root `AGENTS.md` with a stricter bounded-discovery contract.
2. Add key-path references for `.agents/workflows/` and `.agents/templates/`.
3. Create `.agents/workflows/explorer.md` with budgets, stop conditions, output format, and anti-patterns.
4. Create three templates for discovery, shortlist comparison, and ambiguity resolution.
5. Record the rationale and resulting file layout in a workflow report.

## Completion notes

- Root `AGENTS.md` now defines explorer as bounded candidate discovery by default.
- `.agents/workflows/explorer.md` now holds the operational explorer procedure.
- `.agents/templates/` now contains compact task-shaping templates.
- The change keeps stable policy in `AGENTS.md` and moves more volatile operating detail into `.agents/`.
- Template filenames now follow the root convention `.agents/templates/<agent_name>_*.md`.
