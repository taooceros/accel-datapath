# Per-section completion tracking in current.md

## Goal

Refine the `current.md` workflow so completed items are tracked within each thread section rather than in one global completed list.

## Scope

- update root `AGENTS.md`
- restructure root `current.md`

## Planned changes

1. Require each thread section in `current.md` to track its own completed items.
2. Remove the need for a global completed-items section.
3. Update the TOC and section layout in `current.md` to match the per-section model.

## Completion notes

- Root `AGENTS.md` now requires work items to be tracked within each thread section.
- `current.md` no longer uses a global completed-items section.
- Each thread now carries its own completed-items subsection.
