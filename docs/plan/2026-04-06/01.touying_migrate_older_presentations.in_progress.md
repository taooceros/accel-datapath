## Design goals

- keep each deck in its dated directory and preserve the existing deck topic/story
- prefer the shared `deck` wrapper and existing reusable helpers over bespoke page-level formatting
- keep deck-local color aliases or one-off layout helpers local unless they clearly belong in the shared template
- migrate manual `#pagebreak()` slide boundaries toward Touying-native structure where practical
- preserve readability on 16:9 slides; do not force heading-driven structure if it makes dense slides materially worse
- avoid broad template churn during parallel migration; make per-deck changes first, then fill small shared gaps if needed

## Story spine

1. keep the recent interview deck as the reference shape for the shared Touying workflow
2. migrate the most recent older decks first so the shared authoring style works across the active presentation history
3. preserve content fidelity while replacing manual page setup and slide chrome with the shared Touying wrapper
4. compile each deck and fix layout or Typst issues introduced by the migration
