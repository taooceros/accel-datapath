# idxd-rust naming convention cleanup

## Goal

Keep the constructor cleanup readable by using the same names for the same retry concepts. `max_page_fault_retries` should describe the DSA page-fault retry limit, while `submission_retry_budget` should describe direct ENQCMD rejection retries.

## Scope

- Rename host-free test functions that still use generic `retry_budget` wording for `DsaConfig` page-fault retry behavior.
- Preserve direct async names that intentionally use `submission_retry_budget`.
- Update the lean `bon`/`snafu` guard so it checks the current builder normalization path rather than the removed `DsaConfig::with_retries` forwarding helper.
- Do not change runtime behavior, public artifact field names, or direct async failure metadata.

## Intended Changes

1. Rename page-fault retry tests to use `page_fault_retries` wording.
2. Rename the direct async config-preservation test to clarify it preserves the page-fault retry limit, not ENQCMD submission retry budget.
3. Update the guard literal in `idxd-rust/scripts/check_m010_lean_bon_snafu.sh`.
