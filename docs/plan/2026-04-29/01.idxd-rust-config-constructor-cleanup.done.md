# idxd-rust config constructor cleanup

## Goal

Keep the `idxd-rust` DSA configuration API lean by using the existing method-level builder as the single validated `DsaConfig` construction path. This follows the recent lean `bon` convention: keep the builder when it marks the normalization boundary, but avoid duplicate helper constructors that only forward into the same logic.

## Scope

- Preserve the current `DsaConfig::builder().build()` and explicit `.device_path(...).max_page_fault_retries(...).build()` behavior.
- Remove use of the deleted `DsaConfig::new` and `DsaConfig::with_retries` helpers from `idxd-rust` code and tests.
- Keep `DsaSession::open`, `DsaSession::open_with_retries`, `AsyncDsaSession::open`, and `AsyncDsaSession::open_with_retries` as public session conveniences, but route their config construction through the builder.
- Do not change descriptor, completion, direct async, payload-redaction, verifier, or artifact behavior.

## Intended Changes

1. Update sync and async session opening code to construct `DsaConfig` with the builder.
2. Update `idxd-rust` tests to assert builder behavior instead of legacy constructor compatibility.
3. Search for stale references to the removed helpers.

## Completion Notes

Implemented. `DsaConfig::new` and `DsaConfig::with_retries` were removed as duplicate construction helpers. Sync and async session retry-opening paths now construct `DsaConfig` through the method-level builder, and duplicate constructor-compatibility tests were removed.
