# Direct async runtime constructor cleanup

## Goal

Remove duplicate `DirectAsyncMemmoveRuntime` constructor wrappers that only forward into fallible constructors and panic when no Tokio runtime is available. The direct async runtime already has typed `RuntimeUnavailable` diagnostics, so callers should use the fallible construction path explicitly.

## Scope

- Remove `DirectAsyncMemmoveRuntime::new`.
- Remove `DirectAsyncMemmoveRuntime::with_submission_retry_budget`.
- Keep `DirectAsyncMemmoveRuntime::try_new` for default retry-budget construction.
- Keep `DirectAsyncMemmoveRuntime::try_with_submission_retry_budget` for tests that need bounded ENQCMD rejection behavior.
- Update host-free direct async contract tests to call the fallible constructors and preserve their assertions.

## Intended Changes

1. Delete the two panicking wrapper constructors from `idxd-rust/src/async_direct.rs`.
2. Replace test calls to `new` and `with_submission_retry_budget` with `try_new(...).expect(...)` and `try_with_submission_retry_budget(...).expect(...)`.
3. Search for stale constructor references.

## Completion Notes

Implemented. `DirectAsyncMemmoveRuntime::new` and `DirectAsyncMemmoveRuntime::with_submission_retry_budget` were removed. Runtime construction is now explicitly fallible through `try_new` and `try_with_submission_retry_budget`, preserving typed `RuntimeUnavailable` diagnostics instead of hiding them behind panicking convenience wrappers.

The direct async contract tests now use small host-free helpers that call the fallible constructors and keep each test focused on completion, retry, backpressure, shutdown, or payload-redaction behavior.
