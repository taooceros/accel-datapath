# Plan: Improve Mosaic Dev Auto Reload

**Date**: 2026-04-02

**Status**: cancelled on 2026-04-03; superseded by the Observable Framework rewrite plan.

## Goal

Make local iteration on the Mosaic dashboard fast and predictable by adding an auto-reloading development path, while preserving the existing static artifact build for report publishing.

## Scope

### In scope

- inspect the current `devenv` build/serve flow for the Mosaic dashboard
- add a dedicated development command that uses Vite's live-reload/HMR loop
- ensure the dashboard can still read the repo-local tonic CSV during local development
- update developer-facing docs to distinguish dev iteration from artifact build/serve

### Out of scope

- changing the published dashboard artifact structure
- introducing a larger frontend workspace or extra framework
- redesigning the chart UI itself

## Verification

- confirm the new dev command starts successfully via `devenv`
- confirm the production build still succeeds
- verify the updated docs point users to the correct command for iterative work
