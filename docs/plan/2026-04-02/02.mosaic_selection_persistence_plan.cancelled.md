# Plan: Persist Mosaic Selection State

**Date**: 2026-04-02

**Status**: cancelled on 2026-04-03; superseded by the Observable Framework rewrite plan.

## Goal

Persist the Mosaic dashboard's filter and brush selection state in browser localStorage so local reloads keep the user's current exploration context.

## Scope

### In scope

- inspect Mosaic/VGPlot selection APIs for observing and restoring selection state
- add client-side localStorage persistence for the dashboard selection object
- restore saved selection state on page load when compatible
- keep the existing static artifact build working without server-side dependencies

### Out of scope

- syncing selection state across browsers or users
- encoding state in the URL
- redesigning existing controls or charts

## Verification

- production build succeeds after the change
- saved selection state is serialized/restored without breaking initial load
- implementation degrades safely if localStorage is unavailable or saved state is stale
