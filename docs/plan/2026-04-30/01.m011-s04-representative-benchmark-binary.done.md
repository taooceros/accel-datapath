# M011 S04 representative benchmark binary plan

## Goal

Add the small `idxd_representative_bench` binary and host-free contracts for task M011/S04/T01. The binary should exercise the public `IdxdSession<Dsa>` and `IdxdSession<Iax>` representative operations, measure a fixed loop with preallocated buffers, and emit stable no-payload text or JSON artifacts.

## Scope

- Add `idxd-rust/src/bin/idxd_representative_bench.rs` as a consumer-level CLI.
- Add `idxd-rust/tests/idxd_representative_bench_cli_contract.rs` covering CLI validation, failure artifacts, stdout/artifact agreement, schema fields, metric semantics, and redaction.
- Extend `idxd-rust/tests/shared_lifecycle_contract.rs` with source guards that keep the benchmark on the generic session surface and away from raw lifecycle/portal APIs.

## Intended Changes

- Parse `--dsa-device`, `--iax-device`, optional `--dsa-shared-device`, `--bytes`, `--iterations`, `--format text|json`, and `--artifact` without introducing new dependencies.
- Open each configured `IdxdSession<Accel>` once, run one warmup, then time the requested iterations using `Instant`.
- Preallocate deterministic source/destination buffers outside timed loops, use the same zero-residue IAX CRC64 input strategy as `live_idxd_op`, and compare IAX results with `idxd_sys::crc64_t10dif_field`.
- Emit a compact schema with top-level suite/profile/clock/failure metadata and per-target rows for counts, latencies, rates, status/retry/CRC verification, verdict, and failure metadata.
