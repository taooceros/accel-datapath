# Plan: Rewrite Mosaic Report App with Observable Framework

**Date**: 2026-04-03

## Goal

Replace the current custom Vite/Mosaic app with a pure Observable Framework project so the report visualization is structured around Framework pages and small helper modules instead of a hand-built HTML/DOM shell, while managing the app with Bun rather than a Nix-packaged frontend build.

## Scope

### In scope

- remove the current Vite-specific app structure under `tools/mosaic-tonic-report/`
- scaffold an Observable Framework project in the same location
- rebuild the bounded tonic dashboard around Framework markdown pages and Framework-managed static output
- keep the existing bounded tonic matrix CSV as the data source
- update `devenv.nix` so it exposes Bun/tooling without packaging the frontend app as a Nix build product
- keep the published artifact path at `docs/report/artifacts/003.tonic_bounded_matrix_mosaic/`

### Out of scope

- preserving the old DOM assembly pattern or Vite-specific implementation details
- reintroducing every prior UI behavior on the first pass if it complicates the rewrite
- changing the tonic benchmark schema or results format

## Proposed approach

1. Replace the app package/config with Observable Framework equivalents.
2. Re-author the dashboard as `src/index.md` plus small local JS helpers only where markdown becomes unwieldy.
3. Start from a cleaner baseline interaction model and defer nonessential persistence behavior unless it is straightforward in the new structure.
4. Build static Framework output via Bun scripts and continue copying it into the existing report artifact directory.

## Verification

- Bun dependencies install successfully
- local build succeeds for `tools/mosaic-tonic-report`
- the Bun-driven artifact build refreshes `docs/report/artifacts/003.tonic_bounded_matrix_mosaic/`
- diagnostics/build output show no blocking errors in edited files
