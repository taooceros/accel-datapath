## Convention to add

Paper-oriented literature material should live under:

`docs/report/literature/papers/<paper-slug>/README.md`

Where:

- `<paper-slug>` is stable kebab-case derived from the canonical paper title,
- the folder is the long-lived paper home,
- and `README.md` is the KB-searchable paper entry point.

The convention should stay distinct from `docs/related_work/`, which remains topic-oriented.

## Required paper template

Each seeded paper folder should include a `README.md` with these sections:

1. paper identity
2. why it matters here
3. problem
4. method / mechanism
5. evaluation setup
6. key findings
7. limits and confidence
8. repo takeaways
9. source links within repo

The writeup should summarize only what is already supported by local repo artifacts.

The next pass extends the paper folders from explanation-only pages to paper-ingestion homes.

- one searchable Markdown extraction (`paper.md`) derived from that PDF,

Recommended artifact names inside each paper folder:
