# Literature paper folder expansion

**Date**: 2026-04-13
**Status**: completed
**Scope**: expand the per-paper literature folder workflow beyond the seeded six folders to all literature-report papers that already have local PDFs in `papers/top_tier_pdfs/`.

## Goal

Create stable paper folders under `docs/report/literature/papers/` for the broader literature-report corpus, copy the locally downloaded PDFs into those folders, generate searchable `paper.md` artifacts with the reusable extractor, and add `README.md` entry points that connect each paper back to the existing literature summaries.

## In scope

- create paper folders for literature-report papers with local PDFs already present in `papers/top_tier_pdfs/`
- copy each local PDF into the matching paper folder as `paper.pdf`
- generate `paper.md` with `tools/paper-text/extract_paper_text.py`
- keep page-based markdown when sectionization is not yet reliable
- create paper-folder `README.md` files using the existing folder convention and the curated report summaries as source material
- update literature metadata so the expanded folder set is navigable

## Out of scope for this pass

- papers referenced in reports but lacking a local PDF artifact
- deep cleanup of noisy extraction artifacts
- figure-caption cleanup
- broad citation normalization outside the new folder/index updates

## Execution plan

1. Create new paper folders and copy local PDFs into each folder as `paper.pdf`.
2. Run the reusable extractor over each new `paper.pdf` to produce searchable `paper.md`.
3. Create `README.md` entry points from the existing literature report summaries, including local artifact links.
4. Add a new literature index report capturing the expanded folder set and blocked citation-only references.
5. Verify folder completeness, record the result in the matching canonical literature thread file, then refresh the `current.md` dashboard entry.

## Risks

- some PDFs may be lower quality or have two-column extraction noise
- the existing verifier is too strict for arbitrary papers, so page-based extraction is the safe baseline
- title normalization must stay consistent with the folder slug rule used by the seeded set

## Success criteria

- every in-scope local PDF has a matching paper folder with `README.md`, `paper.pdf`, and `paper.md`
- each new README links back to the relevant literature report(s)
- a literature index/report documents what was expanded and what remains blocked by missing PDFs
