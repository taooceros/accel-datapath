# `devenv shell` failure analysis

## Summary

`devenv shell` fails before entering the shell. The failure is not in the
repo's `enterShell` hook or helper scripts. It happens while Nix is realizing
the shell dependency closure from [devenv.nix](/home/hongtao/accel-datapath/iax-hw-eval/devenv.nix), specifically the Python visualization stack ending at
`python3Packages.plotly`.

The observed failure chain is:

`python3.13-plotly` -> `python3.13-scikit-image` -> `python3.13-imageio` ->
`python3.13-pillow-heif` -> `opencv` -> `cuda12.8-cuda_compat`

The lowest-level error is:

```text
cuda12.8-cuda_compat-12.8.39468522.drv: variable $src or $srcs should point to the source
```

Because that derivation fails, `devenv` cannot realize the shell derivation.

## What I checked

### Prior project notes

No existing report mentions a prior `devenv shell` failure or a known workaround.

### Expected entrypoint

[README.md](/home/hongtao/accel-datapath/iax-hw-eval/README.md) directs users to enter the
development environment with:

```bash
devenv shell
```

### Shell definition

[devenv.nix](/home/hongtao/accel-datapath/iax-hw-eval/devenv.nix) includes these Python packages
in the default shell:

- `python3Packages.pandas`
- `python3Packages.matplotlib`
- `python3Packages.numpy`
- `python3Packages.plotly`

### Reproduction

Command:

```bash
devenv shell -- true
```

Observed tail of the error:

```text
error: Cannot build '/nix/store/...-cuda12.8-cuda_compat-12.8.39468522.drv'.
Reason: builder failed with exit code 1
...
Running phase: unpackPhase
variable $src or $srcs should point to the source
...
error: Cannot build '/nix/store/...-python3.13-plotly-6.6.0.drv'.
Reason: 1 dependency failed
...
Error:
× Evaluation error: Failed to realize shell derivation
```

### Local nixpkgs config check

There is no `~/.config/nixpkgs` directory on this machine, and an impure
`<nixpkgs>` import reports:

```nix
{ allowUnfree = false; cudaSupport = false; }
```

So this does not look like a user-local `cudaSupport = true` override.

## Root cause

The default `devenv` shell currently pulls in a Python package closure that is
broken in the active `devenv` / nixpkgs package set on this machine. The first
high-level package exposed by the repo config that shows up in the failing chain
is `python3Packages.plotly`.

That means:

1. `devenv shell` is failing during dependency realization.
2. The failure is upstream of the repo's own shell hooks.
3. The most likely repo-local trigger is the inclusion of `plotly` in the
   default shell package list.

## Practical fixes

### Lowest-risk repo-local fix

Remove `python3Packages.plotly` from the default shell in
[devenv.nix](/home/hongtao/accel-datapath/iax-hw-eval/devenv.nix), or move visualization packages into an
optional shell/profile instead of the base development shell.

### If `plotly` is required

Possible alternatives:

1. Pin or override the Python package set to a nixpkgs revision where
   `plotly` realizes cleanly.
2. Override the problematic package path in nixpkgs rather than carrying it in
   the default shell.
3. Split plotting tools into a separate environment so core development does not
   depend on that closure.

## Minimal conclusion

`devenv shell` fails because the default shell depends on
`python3Packages.plotly`, and the current package closure for that dependency
fails while building `cuda12.8-cuda_compat`. The shell never gets far enough to
run the repo's own shell initialization.
