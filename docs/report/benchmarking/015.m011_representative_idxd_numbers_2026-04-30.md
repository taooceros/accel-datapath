# M011 S04 representative IDXD measured numbers (2026-04-30)

## Summary

S04 produced the small release-profile measured-number proof required by R023 for the representative generic `IdxdSession<Accel>` paths. The proof used the new `idxd_representative_bench` verifier, not `tokio_memmove_bench`, S03 `live_idxd_op`, `live_memmove`, `hw-eval`, a debug build, or host-free fixtures.

Closure result:

- `dsa-memmove:/dev/dsa/wq0.0` passed through `IdxdSession<Dsa>` with positive latency and rate metrics.
- `iax-crc64:/dev/iax/wq1.0` passed through `IdxdSession<Iax>` with positive latency and rate metrics and `crc64_verified=true`.
- Top-level verifier evidence ended with `verdict=pass`, `launcher_status=ready`, `profile=release`, and `claim_eligible=true`.

This report is a bounded representative proof. It is not a benchmark matrix, not a full performance characterization, and not a final performance comparison.

## No-payload and scope boundary

The no-payload contract remains in force: this report records operation names, accelerator family, device paths, WQ mode, requested byte count, iteration count, timing aggregates, retry counts, status values, CRC scalar values, and artifact paths only. It does not include raw buffer bytes, descriptors, tokens, secrets, or byte dumps.

The claim boundary is intentionally narrow:

- This is one tiny representative suite with the required DSA memmove and IAX/IAA crc64 rows.
- It does not sweep sizes, queue modes, concurrency, batching, or software baselines.
- It does not claim optional shared DSA because `IDXD_RUST_VERIFY_DSA_SHARED_DEVICE` was not configured for this run.
- Expected-failure output from an unprepared host would remain diagnostic only and would not satisfy R023.
