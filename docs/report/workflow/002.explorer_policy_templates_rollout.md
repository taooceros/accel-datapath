# Explorer policy and templates rollout

**Date**: 2026-04-01
**Scope**: root `AGENTS.md`, `.agents/workflows/explorer.md`, and `.agents/templates/explorer_*.md`

## Summary

This change turns the explorer workflow into a two-layer system:

- root `AGENTS.md` carries short, stable policy
- `.agents/` carries procedural detail and reusable task templates

The goal is to reduce `@explorer` context blowups by making bounded discovery the default shape and by making prompt structure explicit.

## Changes made

### 1. Tightened root explorer policy

`AGENTS.md` now says that `@explorer` should:

- do bounded candidate discovery by default
- avoid combining repo-wide discovery, broad reading, and final recommendation in one task
- receive explicit objective, scope boundary, allowed sources, max candidates, and stop condition
- obey default limits for search rounds, domains, candidates, and reads per candidate
- stop early and hand back when candidate discovery is good enough or no longer converging

It also adds `.agents/workflows/` and `.agents/templates/` to the repo key paths.

### 2. Added an explorer workflow guide

`.agents/workflows/explorer.md` now captures the operating procedure that did not belong in `AGENTS.md`:

- allowed task shapes
- standard budget
- stop conditions
- compact output schema
- anti-patterns that tend to trigger context growth

### 3. Added reusable prompt templates

Three template files were added:

- `.agents/templates/explorer_discovery.md`
- `.agents/templates/explorer_compare.md`
- `.agents/templates/explorer_disambiguate.md`

These make it easier for the orchestrator to delegate small, bounded explorer jobs instead of broad “understand X” requests.

## Why this structure

Keeping all details in `AGENTS.md` would make the main policy file too dense and easier to ignore. Splitting the content gives:

- **durability** in `AGENTS.md` for rules that should hold every session
- **flexibility** in `.agents/workflows/` for procedures that may evolve
- **reuse** in `.agents/templates/` for prompt shaping without repeating instructions ad hoc

The root policy now also defines the naming convention `.agents/templates/<agent_name>_*.md` and requires delegations to cite a specific matching template path instead of referencing the template directory generically.

## Expected impact

The expected workflow shift is:

1. explorer discovers candidates
2. orchestrator prunes and chooses follow-up questions
3. a later bounded task does selective deepening if needed

This should reduce the chance that explorer sessions carry too many files, hypotheses, and snippets long enough to trigger auto compaction.
