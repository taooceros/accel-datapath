# `hw-eval` review-fix validation (2026-03-23)

## Summary

Validated the two follow-up fixes for the `hw-eval` IAX backend:

1. `--accel iax` now resolves the default device to `/dev/iax/wq1.0` when
   `--device` is omitted.
2. The sliding-window throughput path runs successfully after the submission
   accounting fix and reports stable scaling across the requested concurrency
   sweep.

## Commands run

Build:

```bash
cd /home/hongtao/accel-datapath/iax-hw-eval
devenv shell -- bash -lc 'cd hw-eval && cargo build --release'
```

Minimal smoke test using the new default IAX device selection:

```bash
devenv shell -- bash -lc \
  'timeout 30s ./tools/build/dsa_launcher ./hw-eval/target/release/hw-eval \
     --accel iax --iterations 10 --sizes 64 --max-concurrency 4 --json'
```

Larger validation sweep:

```bash
devenv shell -- bash -lc \
  'timeout 60s ./tools/build/dsa_launcher ./hw-eval/target/release/hw-eval \
     --accel iax --iterations 1000 --sizes 64,256,1024 --max-concurrency 8 --json'
```

## Results

### Build

`cargo build --release` succeeded for `hw-eval`.

### Default-device fix

The smoke test succeeded without passing `--device`, and the JSON metadata
reported:

```json
"accelerator": "iax",
"device": "/dev/iax/wq1.0"
```

This confirms the runtime now selects the IAX work queue by default for
`--accel iax`.

### Sliding-window throughput sweep

The larger run completed successfully for sizes 64, 256, and 1024 bytes with
concurrency sweep 1, 2, 4, 8.

## Interpretation

- The default-device regression is fixed: the IAX path now opens
  `/dev/iax/wq1.0` without an explicit `--device` argument.
- The corrected sliding-window path completes normally under a larger sweep and
  shows expected throughput scaling as concurrency increases from 1 to 8.
- This validation does not include a pre-fix benchmark capture, so it confirms
  post-fix behavior only.
